# 유형테스트 설계 알고리즘 (근거 기반)

"대충 만든 티가 나지 않고, 테스트한 사람에게 신뢰를 주는" 테스트를 자동
생성하기 위한 설계 명세. 2020~2026년 국내외 히트 테스트 케이스와 심리측정
문헌을 딥리서치한 결과를 규칙으로 압축했고, `src/quiz/`의 스키마·검증·채점·
렌더링이 이 명세를 코드로 강제한다.

## 근거 요약 — 왜 이 구조인가

### 채점: 단순 합산(argmax) 금지, 축 기반 스펙트럼 채점

- 성격은 자연적 '유형'이 아니라 연속 차원으로 분포한다 — Big Five가 MBTI보다
  재검사 신뢰도(r=0.75~0.90 vs ~0.50)에서 우월한 이유. 축별 퍼센트 표기는 이
  과학적 관례를 차용해 신뢰감을 만든다.
- 16personalities(누적 10억+ 응시)의 핵심 장치가 정확히 이것: 이분법 대신
  "Introverted 64%" 스펙트럼 바. 같은 유형이라도 바 모양이 달라 "나만의
  결과"라는 개인화 지각이 생기고, 경계선(52:48) 사용자의 반발을 흡수한다.
- 단순 유형 합산은 사용자가 답→결과를 역추적할 수 있어 투명하게 얕아 보이고,
  동점 처리가 자의적으로 느껴진다. 국내 히트작 중에도 푸망 표준 포맷(3축×
  3문항→8유형), 신한카드 소BTI(4축→16유형), SPTI(4축, 3주 400만 참여)처럼
  축 조합형이 상위권 표준.

**구현**: 축 2~4개, 답변마다 극 방향(left/right)+가중치(1~2), 축 점수 =
좌극점수/(좌+우) → 0~100%, 유형 = 지배 극 코드 조합(2^축수). 50:50은 left로
결정(결정성 보장, 축당 홀수 문항 권장으로 희귀화). → `src/quiz/engine.js`

### 문항: 상황 제시형, 정답 냄새 제거, 역채점 균형

- 상황판단형 문항은 직접 자기보고보다 꾸밈·사회적 바람직성에 강하고 몰입도가
  높다. 국내 히트작의 공통 문법("애인 입가에 음식이 묻었다. 당신의 행동은?" —
  연애능력 테스트, 5일 200만 참여).
- 사회적으로 바람직한 답이 하나뿐인 문항은 실패작 — 모든 선택지가 각자 매력
  있거나 각자 웃겨야 한다.
- 같은 축의 문항들이 전부 같은 방향을 밀면(1번 답만 누르면 특정 유형) 조작
  가능성이 보인다 — 방향을 섞는 역채점 균형 필수.

**구현**: 문항당 1축 강제, 답변에 양극 모두 존재해야 검증 통과, 프롬프트에
상황 제시형·정답 냄새 금지 규칙 명시. → `validateQuiz`, `buildPrompt`

### 분량: 9~13문항, 1~3분, 유형 8~16개

- 완주율 데이터: 3~7문항 65~85% vs 16문항+ 25~45%로 급락. 국내 히트작
  스위트스팟도 9~13문항(SPTI 12, 연애능력 13, 푸망 표준 9).
- 신뢰감을 위한 하한: 축당 3문항 이상은 있어야 판정이 안정적으로 보인다.
- 유형 수는 구조가 설명되는 4/8/16이 표준. 3개면 성의 없어 보이고 근거 구조
  없는 30개는 랜덤처럼 느껴진다.

**구현**: 총 8~15문항, 축당 3문항 이상, 유형 수 = 2^축수 강제. AI 생성
기본값은 3축(9~12문항→8유형).

### 결과 페이지: 신뢰 장치를 전부 싣는다

리서치에서 확인된 신뢰 장치와 구현 위치:

| 장치 | 근거 | 구현 |
|---|---|---|
| 축별 퍼센트 바 | 16personalities 핵심 장치. "측정했다"는 인상 | 결과 페이지 `?p=` 개인 퍼센트 렌더 |
| "응답자 중 N%" 희소성 통계 | MBTI 희귀유형 담론, 캐릿 트능 "상위 N%", Wrapped 퍼센타일 | 실응답 누적 집계 + 라플라스 스무딩 (`QuizStore.recordResponse`) |
| 강점 80 : 약점 20 | 칭찬만 있는 결과는 가짜같이 느껴짐(Forer 연구의 경고 신호). 국내 분석은 "칭찬 7:뼈 3" | 강점 3~5 + 성장 포인트 1~2 검증 강제 |
| 궁합(케미) | 국내 확산 계수의 핵심 — "네 것도 해봐"를 구조적으로 강제 | bestMatch/worstMatch 필수, 결과 페이지 케미 카드 |
| 실행 가능한 조언 | 꼰대성향검사 "탈출법", 연애능력 "강화 팁" — 처방이 "용하다" 체감을 만듦 | advice 1~3개 강제 |
| 결과 카드 심미성 | 케이테스트 퍼스널컬러(200만 참여)의 승부처 — "스토리에 올려도 안 부끄러운가" | 유형별 고정 색상 + 스크린샷 완결형 카드 |
| 정체성 언어 유형명 | "73점"이 아니라 부를 수 있는 별칭(테토남처럼 어휘가 밈이 되면 재유입) | 프롬프트 규칙 |
| 면책 라벨 | "재미로 보는" 명시가 과학 참칭 역풍(테토-에겐 사례)을 예방, 신뢰도 깎지 않음 | 전 페이지 fineprint |
| 결과-응답 연결 | BuzzFeed 전성기 공식 "왜 이 결과인지 설명" | 개인 퍼센트 바가 곧 근거 표시 |

### 공유 루프

- 결과 페이지가 공유 단위: 결과별 고유 OG 타이틀("나는 ○○!") → 미리보기에서
  결과가 보이고 CTA로 재유입. 공유 링크에는 개인 퍼센트가 없어서(의도적)
  "직접 해보면 내 퍼센트가 나온다"가 참여 훅이 된다.
- shareText는 "I got ___" 템플릿 — 긍정 단어가 든 결과가 실제로 더 공유된다.

### 안티패턴 (검증이 막는 것들)

- 답이 빤한 문항 매핑(조작 가능성 노출) → 극 혼합 검증
- 두 줄짜리 결과문 / 100% 칭찬 바넘 → 강점·약점·조언 개수 강제
- 유형 조합 누락·중복, 자기 자신과의 궁합 → 커버리지 검증
- 결과 직전 개인정보 요구 → 없음(즉시 결과 공개)
- 정치/종교/성인 소재(광고 및 논란 리스크) → 토픽 선별 단계에서 제외

## 케이스 레퍼런스 (리서치 출처 발췌)

- **국내**: 테토-에겐(2025 구글 올해의 검색어), SPTI 스낵 성격테스트(3주
  400만), 연애능력 테스트(5일 200만), 꼰대성향검사 KKDTI(누적 300만, 대학내일
  20대연구소 자문), 캐릿 트렌드 능력고사(연 200만), 케이테스트 퍼스널컬러
  (200만+, 결과 카드 심미성의 원형), 푸망(누적 7,500만 플레이, 3축×3문항
  표준), 신한카드 소BTI(MBTI 축의 도메인 번안), 오늘의집 전생집(결과의 구체성).
- **글로벌**: 16personalities(축+퍼센트+물량형 결과 페이지+인구통계),
  uQuiz Human Feeling(작가성·시적 결과문), ktestone Smile Dating(결과 카드
  디자인), IDRlabs(학술 인용+그래프+면책 병치), Spotify Wrapped(퍼센타일
  자랑, 행동 데이터 서사), BuzzFeed(흥망 — 양산형·AI 티가 나는 순간 몰락).
- **방법론**: Big Five vs MBTI 신뢰도 비교, Barnum/Forer 효과 연구, 상황판단형
  문항(SJT) 연구, 역채점·묵인편향 연구, 완주율-문항수 데이터(SurveyMonkey/
  Outgrow/Interact).

원 리포트의 상세 출처 URL은 PR 논의 스레드 참고. 규칙이 코드 어디에
반영됐는지는 `src/quiz/generate.js`(스키마·프롬프트·검증),
`src/quiz/engine.js`(채점), `src/quiz/render.js`(결과 페이지),
`src/quiz/store.js`(통계)에 주석으로 남겨두었다.

## 향후 확장 (리서치가 제안하는 다음 단계)

1. **일관성 배지**: 같은 축의 유사 문항 2개 응답이 일치하면 "응답 일관성 높음"
   배지 — 정식 검사의 validity scale을 축소 차용한 신뢰 장치.
2. **결과 카드 이미지 생성**: OG 이미지를 유형별 색상 카드로 동적 생성
   (현재는 공용 아이콘) — 케이테스트급 공유율의 핵심.
3. **프레임 네이밍**: 주차별 테스트에 고유 프레임 이름(예: "위클리 이슈 반응
   모델")을 부여해 16personalities의 NERIS처럼 브랜드화.
4. **행동 데이터 연동**: 피드 사용 기록 기반 결과(Wrapped형)는 자기보고보다
   정당성 체감이 높다 — 피드 사용자 대상 "내 피드 취향 리포트"로 확장 가능.
